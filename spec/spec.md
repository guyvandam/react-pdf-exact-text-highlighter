# main

This is a package for creating exact-text-highlighting links in pdfs (and later other formats)

## 2 important aspects

1. the package logic itself
2. manual testing curation using the test-case-harness

## 1. logic itself

A pdf viewer with exact-text-highlighting links functionality

- enables the user to create highlights that are links to that highlight in the pdf.
  - whenever that link is pressed, the reader navigates to that highlight
  - upon reaching that hihglight it shows a quick animation which is a halo-like animation around the highlight box
- it uses the `react-pdf-highlighter-extended`package [>](https://github.com/DanielArnould/react-pdf-highlighter-extended?tab=readme-ov-file)
- each highlight, upon invokation, should return if it was found in the pdf and showed to the user. i.e. - was it sucessful? was there any weird characters in that blocked selection?

## 2. testing curation harness

The second pilar is the manual curation harness - which is used for creating test cases for said highlights.

- has a pdf selector
- viewer
- highlighting options

upon creating a highlight, it automatically gets appended to the tests.

- when pressing each highlight test case, it shows the highlight in the pdf.

# LLM integration

A neccsary pillar for this package - *the ability for LLMs to accuratly cite pdfs*

the pdf will be loaded into an agent LLM's context w/ the cite tool which enables it to create the airtight citation in this react package!

---

## testing strategy

### creating the LLMs test cases

1. specify prompt(s) for the LLM
  - resides in separate markdown files
2. create test-creation run (w/ its own id)
3. loops over all pdfs in `/tests` folder
4. for each pdf, have the LLM agent (e.g. claude code) generate a response for each pdf - with a lot of citations and links back to the pdf.
## output format
1. each ai tests generation has its own folder and id
2. each pdf own folder
  - each prompt its own output trajectory - the whole trajectory made by the LLM agent (citation calls, retries, etc..)
3. each run it's own `expectation.json`, following the manual testing format!

## running the tests
1. run the main manual `expectation.json`
2. run all existing ai test expectations (generated by the ai)
